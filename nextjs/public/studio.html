01Start template▶
Start from a template, then shape it. The 2D→3D engine works for any object you trace — these are car-frame starting shapes, and more object templates are coming. Everything below updates the 3D model live.
02Reference drawing▶
Trace a side view for the profile, a top view for real width, and front / rear views for the end cross-sections. Switch views in the Reference & Trace tab. All optional — the sliders work without any drawing.
Side scale not set · Top scale not set
03Silhouette▶
Source: parametric — trace in the 2D tab to override.
04Width▶
Source: parametric — trace a top view to derive width.
Source: parametric — a front view sets the real cross-section.
05Wheel cutouts · car frame▶
Car-frame feature: arches are cut into the shell to clear the wheels. Set these to the chassis you're designing around. (Wheels and chassis will become their own traceable objects you assemble — for now this just shapes the frame.)
Diameter = length gives a round wheel; make them differ for a stretched/oval wheel.
0 = wheels don't shape the frame at all. Higher = deeper arches cut to clear the wheels.
06Shell & mesh▶
Smooth loft: stretches the front section along the length — softer, approximate. Orthographic: intersects the side + front + top silhouettes so every line projects to its true place (needs a front view).
★Section cuts & 3D edit▶
A cross-section is pinned at each cut; the model morphs between them. Front & rear views become cuts automatically. Drag the dots to reshape the selected cut — this edits the 3D model live.
No section cuts yet — trace a front view, import a sheet, or add one.
07Export & library▶
With LEE3D-Backend-A running, these call it directly: Save commits this profile into LEE3D-Lib; Build on server runs the CadQuery pipeline (true wheel openings + shelling) and downloads a STEP you can keep editing in any CAD tool. No backend? Both fall back to a local download. serve the studio from localhost:3000 or add your origin to the backend CORS list
☁Cloud database · Supabase▶
Save projects to a Supabase (Postgres) database so they persist and sync across devices — no backend server to run. Paste your project URL + anon public key. ▸ show setup SQL
